DynamoDB Student Records Demo

Hands-On AWS Console Workshop

Demo Overview

This hands-on demonstration will guide you through building a complete DynamoDB solution using the AWS Management Console. You will work with a StudentRecords database to understand core DynamoDB concepts through practical application.

What We Will Build

A student records management system that demonstrates fundamental DynamoDB operations including table creation, data modeling with primary keys and secondary indexes, query optimization, and data protection through backup strategies.

Learning Approach

Each step builds upon the previous one, starting with basic table structure and progressing through advanced querying techniques. You will see immediate results in the AWS console, reinforcing theoretical concepts with practical experience.

2

Learning Objectives

By completing this demonstration, you will gain hands-on experience with essential DynamoDB operations and understand how to implement them in real-world scenarios.

  • Create and configure DynamoDB tables with appropriate primary key structures for your data access patterns
  • Design and implement Global Secondary Indexes to enable additional query capabilities beyond the primary key
  • Execute efficient Query operations using primary keys and understand when to use Scan operations with filters
  • Utilize PartiQL to perform SQL-like queries on both base tables and secondary indexes
  • Configure Point-in-Time Recovery for data protection and compliance requirements
  • Implement proper resource cleanup to manage costs and maintain account hygiene

Prerequisites and Setup

AWS Account Access Active AWS account with access to the Management Console and DynamoDB permissions. Your IAM user or role must have dynamodb:* permissions to complete all demonstration steps.
Browser Requirements Modern web browser such as Chrome, Firefox, Safari, or Edge for optimal AWS console experience and compatibility with all demonstration features.
Conceptual Knowledge Basic understanding of database concepts including tables, keys, and indexes. Familiarity with NoSQL concepts is helpful but not required.

Initial Setup Steps

  • Navigate to the AWS Management Console at console.aws.amazon.com
  • Verify your IAM user has appropriate DynamoDB permissions
  • Select your preferred AWS region in the top-right corner of the console
  • Bookmark the DynamoDB service page for quick access during the demonstration

Step 1: Create Table and Global Secondary Index

Create the StudentRecords Table

  • Open DynamoDB service from the AWS Console search bar
  • Navigate to Tables and click Create table
  • Enter table name: StudentRecords
  • Set partition key: StudentId (String type)
  • Set sort key: CourseName (String type)
  • Select Use default settings for initial configuration
  • Click Create table and wait for Active status

Add Global Secondary Index

  • Open your StudentRecords table
  • Navigate to the Indexes tab
  • Click Create index
  • Set partition key: CourseName (String type)
  • Set sort key: Grade (Number type)
  • Index name: CourseGrades-GSI
  • Accept default settings and create the index
Design Note: The composite primary key (StudentId + CourseName) enables efficient student-specific queries, while the GSI (CourseName + Grade) allows course-based queries and grade ranking.

Step 2: Data Operations and Querying

Create Sample Data

Add three student records to demonstrate different query patterns:

StudentId CourseName Grade Attendance Credits
S101 Mathematics 85 90 3
S102 Mathematics 92 95 3
S101 Physics 88 85 4

Query Operation

  • Navigate to Explore table items
  • Select Query operation
  • Set StudentId = S101
  • Execute to see all courses for student S101

Scan with Filter

  • Select Scan operation
  • Add filter: Grade > 90
  • Execute to find high-performing students
  • Observe the difference in operation efficiency

Step 3: PartiQL SQL-like Queries

Base Table Query

Use PartiQL to query the main table with familiar SQL syntax:

SELECT * FROM "StudentRecords" WHERE "StudentId" = 'S101';

This query returns all courses enrolled by student S101, demonstrating how PartiQL provides SQL-like syntax while leveraging DynamoDB's key-based access patterns.

Global Secondary Index Query

Query the GSI to get course-specific data ranked by performance:

SELECT CourseName, Grade, StudentId FROM "StudentRecords"."CourseGrades-GSI" WHERE CourseName = 'Mathematics' ORDER BY Grade DESC;

This query utilizes the GSI to efficiently find all students in Mathematics, automatically sorted by grade in descending order.

PartiQL Benefits: Combines the familiar SQL query syntax with DynamoDB's performance and scalability, making it easier for teams with SQL experience to work with NoSQL data.

Step 4: Update Operations and PITR

Update Student Grade

  • Navigate to the Items tab in your StudentRecords table
  • Locate the item with StudentId = S101 and CourseName = Mathematics
  • Click Actions and select Edit
  • Change the Grade value from 85 to 88
  • Save the changes and verify the update in the table

This demonstrates DynamoDB's ability to efficiently update individual attributes without affecting other data in the item.

Enable Point-in-Time Recovery

  • Select your StudentRecords table
  • Navigate to the Backups tab
  • Click Edit to modify backup settings
  • Enable Point-in-time recovery (PITR)
  • Save configuration and confirm PITR status shows as On

PITR provides continuous backups with per-second granularity for up to 35 days, enabling recovery from accidental data changes or deletions.

Step 5: Resource Cleanup

Delete the StudentRecords Table

  • Navigate back to the Tables view in DynamoDB
  • Select your StudentRecords table from the list
  • Click Actions and select Delete table
  • Type "confirm" in the confirmation prompt
  • Click Delete to permanently remove the table
  • Wait approximately 10 seconds and refresh to verify deletion
Important: Deleting a DynamoDB table permanently removes all data, indexes, and associated settings. This action cannot be undone unless you have backups enabled and available.

Cost Considerations

Key Learnings and Best Practices

Table Design Principles

The composite primary key structure (StudentId + CourseName) demonstrates how to design tables for specific access patterns. This design enables efficient queries for individual students while maintaining uniqueness across student-course combinations.

Secondary Index Strategy

The Global Secondary Index (CourseName + Grade) shows how to enable additional query patterns without duplicating data unnecessarily. The GSI allows course-based queries and automatic sorting by grade, which would be impossible with only the base table structure.

Query Optimization

The demonstration highlights the efficiency difference between Query operations (which use key-based access) and Scan operations (which examine all items). Understanding this distinction is crucial for building performant applications.

Data Protection

Point-in-Time Recovery provides automated backup capabilities without impacting application performance, demonstrating DynamoDB's built-in data protection features for production workloads.
